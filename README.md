# PriceFDM — Калькулятор стоимости 3D-печати

Десктоп-приложение (Windows / macOS) для расчёта себестоимости и продажной цены изделий, напечатанных на FDM-принтере. Помогает определить справедливую цену с учётом всех реальных затрат: расхода материала, времени печати, электроэнергии, износа оборудования и постобработки.

📖 **[Руководство пользователя](https://sphenoid-yam-beb.notion.site/PriceFDM-321f33a594ef808c8946cb214908305d)**

## История версий

### v1.2.1 — апрель 2026

- **Перенос проекта в «Прибыль» одним кликом**: в заголовке каждого проекта в истории появилась зелёная кнопка — она добавляет все расчёты проекта в таблицу прибыли сразу, без необходимости переносить каждую деталь отдельно.
- **Меню View и Window на macOS**: добавлены стандартные пункты View (зум, полноэкранный режим) и Window (свернуть, на передний план) — теперь все системные функции доступны из меню.

- Исправлены мелкие недочёты и баги.

### v1.2.0 — апрель 2026

- **График прибыли**: во вкладке «Прибыль» появилась вкладка «График» — столбчатая диаграмма (recharts) по каждой позиции с тремя барами: Выручка, Себестоимость, Прибыль. При наведении — всплывающая подсказка с точными значениями.
- **Поиск и фильтрация в таблице прибыли**: текстовое поле фильтрует строки по названию позиции в реальном времени.
- **Сортировка столбцов в таблице прибыли**: клик по заголовку любого столбца (название, количество, себестоимость, цена продажи, выручка, прибыль) переключает направление сортировки.
- **Сортировка в истории**: кнопки-переключатели в правом верхнем углу истории позволяют сортировать расчёты по дате, общей цене или названию детали.
- **Комментарий к расчёту**: при сохранении расчёта в историю можно добавить текстовый комментарий. В истории — кнопка «Изменить комментарий» открывает инлайн-редактор; если комментарий есть, он отображается под чипами расчёта.
- **Кошт-лист**: кнопка «📄» на карточке расчёта в истории открывает «Кошт-лист» — оформленный документ со всеми параметрами заказа, разбивкой по статьям и итоговой ценой. Можно скопировать текст в буфер обмена или распечатать в браузере.
- **Симулятор % прибыли**: в блоке результатов расчёта появилась кнопка ⚙ (иконка настройки), которая раскрывает слайдер. Перетаскивание слайдера показывает, какой была бы цена при другом проценте прибыли — без изменения реального расчёта. Для партий отображается итоговая сумма заказа.
- **Статистика во вкладке «Расчёт»**: если в истории есть хотя бы один расчёт, над формой появляются три чипа: количество расчётов, суммарная выручка и суммарная прибыль по всей истории.
- **Онбординг при первом запуске**: при первом открытии приложения показывается диалог-инструкция с тремя шагами: добавить принтер → добавить катушку → сделать расчёт. Каждый шаг содержит кнопку перехода на нужную вкладку. Показывается один раз; можно пропустить.

### v1.1.5 — март 2026

- **Автоподбор катушки из G-code**: при импорте G-code приложение читает тип пластика (`filament_type`), название пресета (`filament_settings_id`), цвет и производителя. Если в списке катушек найдено совпадение по названию пресета или единственная катушка нужного типа — она выбирается автоматически. Если совпадений несколько — выводится подсказка с количеством.
- **Авто-выбор принтера**: если в профилях добавлен только один принтер — он подставляется в форму автоматически без ручного выбора.
- **Подсказка при отсутствии веса модели**: если в G-code нет отдельного веса модели (типично для OrcaSlicer с поддержками) — поле показывает предупреждение с тултипом. Расчёт в этом случае ведётся по полному весу.
  

### v1.1.0 — март 2026

- **Папки-проекты**: расчёты можно группировать по проектам прямо в истории.
- **Один расчёт — несколько папок**: расчёт можно добавить сразу в несколько проектов.
- **Цветовые метки проектов**: каждому проекту назначается цвет из палитры — для быстрой навигации.
- **Учёт процента брака**: в заголовке проекта указывается ожидаемый % неудачных печатей; рядом с общей суммой показывается скорректированная цена.
- **Оптовый заказ**: при количестве > 1 появляется галочка «Оптовый заказ» со скидкой (по умолчанию 10%); результат показывает цену до и после скидки.
- **Импорт из G-code**: кнопка «Импорт G-code» в форме расчёта автоматически берёт вес детали и время печати из gcode-файла (PrusaSlicer, OrcaSlicer, Cura). При наличии поддержек OrcaSlicer/PrusaSlicer разделяют вес модели и суммарный расход — оба поля заполняются автоматически.
- **Импорт катушек из OrcaSlicer**: кнопка «Импорт из Slicer» в разделе «Катушки» читает архив `Filament presets.zip`, экспортированный из Slicer. Из каждого JSON-пресета извлекается название, цвет (`default_filament_colour`), тип пластика (`inherits`) и цена за кг (`filament_cost`). Можно выбрать нужные пресеты из списка и добавить их одним кликом.
- **Несколько собственных этапов обработки**: теперь можно добавить любое количество пользовательских этапов постобработки (кнопка «Добавить свой этап») и удалить ненужные.
- **Автосохранение черновика**: при закрытии приложения незавершённый расчёт сохраняется автоматически и восстанавливается при следующем открытии.
- **Миграция данных**: старые сохранённые расчёты (поле `projectId`) корректно переносятся в новый формат (`projectIds`).
- **Версионирование хранилища**: формат localStorage versioned для упрощения будущих миграций.

## Формулы расчёта

### 1. Стоимость материала

```
цена_граммa = цена_катушки / вес_катушки

стоимость_материала = цена_грамма × расход_филамента
```

> Расход филамента — полный вес с поддержками и юбкой (берётся из слайсера).

---

### 2. Стоимость электроэнергии

```
стоимость_электроэнергии = (мощность_Вт / 1000) × время_печати_ч × тариф_₽/кВтч
```

---

### 3. Износ принтера

```
износ_в_час = стоимость_принтера / ресурс_принтера_ч

стоимость_износа = износ_в_час × время_печати_ч
```

> Отражает постепенную амортизацию оборудования. Рекомендуемый ресурс: 3 000–5 000 ч.

---

### 4. Коэффициент сложности

Применяется к **стоимости износа** и **стоимости постобработки**. Материал и электроэнергию не затрагивает.

| Уровень | Коэффициент | Когда применять |
|---|---|---|
| Очень простая | ×1.0 | Коробки, корпуса, простые детали |
| Обычная | ×1.15 | Умеренные поддержки |
| Сложная | ×1.3 | Много мелких элементов |
| Очень сложная | ×1.6 | Миниатюры, высокий риск брака |

```
скорр_износ       = стоимость_износа × коэффициент_сложности
скорр_обработка   = стоимость_обработки × коэффициент_сложности
```

---

### 5. Постобработка

Каждый этап включается отдельно. Для этапов «за грамм» стоимость вычисляется автоматически по весу модели:

```
стоимость_этапа = ставка_₽/г × вес_модели_г
```

> Вес модели — вес без поддержек (берётся из слайсера отдельно).

Ставки по умолчанию:

| Этап | Режим | Значение |
|---|---|---|
| Снятие поддержек | Фиксированная | 50 ₽ |
| Шлифовка | За грамм | 1 ₽/г |
| Покраска | За грамм | 2.7 ₽/г |
| Склейка | Фиксированная | 50 ₽ |
| Грунтовка | За грамм | 0.75 ₽/г |
| Лакировка | За грамм | 1 ₽/г |
| Упаковка | Фиксированная | 150 ₽ |

Стоимость каждого этапа можно изменить вручную.

---

### 6. Итоговая себестоимость

```
себестоимость = материал + электроэнергия + скорр_износ + скорр_обработка + доп_расходы
```

---

### 7. Продажная цена

Прибыль в процентах:
```
прибыль = себестоимость × процент / 100
```

Фиксированная прибыль:
```
прибыль = заданная_сумма
```

```
цена_за_штуку = себестоимость + прибыль
```

## Как пользоваться

### Первый запуск

1. Откройте вкладку **Профили** → добавьте свои катушки с пластиком (название, тип, цвет, цена, вес катушки)
2. Там же добавьте профиль принтера (мощность в Вт, стоимость, ресурс в часах)
3. Откройте вкладку **Настройки** и укажите ваш тариф на электроэнергию

### Расчёт стоимости

1. Откройте вкладку **Калькулятор**
2. Нажмите **Импорт G-code** — вес, время, катушка и принтер заполнятся автоматически
3. Уточните **название детали** при необходимости
4. Если в модели есть поддержки, введите **вес модели** вручную (поле покажет подсказку)
5. Если катушка не подобралась автоматически — выберите **профиль катушки** вручную
6. Если принтеров несколько — выберите **профиль принтера** вручную
7. Выберите **сложность модели**
8. При необходимости включите этапы **постобработки** — стоимость по весу рассчитается сама
9. Задайте **прибыль** (% или фиксированная сумма)
10. Нажмите **Сохранить в историю**

В блоке **Детализация** вы увидите разбивку по каждой статье расходов с учётом коэффициента сложности.

### История расчётов

После расчёта нажмите **Сохранить в историю**. Любой сохранённый расчёт можно снова загрузить в форму кнопкой восстановления.

## Запуск и сборка

### Установка зависимостей

```bash
npm install
```

---

### Запуск в браузере (веб-режим)

Удобен для разработки или быстрого использования без установки:

```bash
npm run dev
```

Открыть: [http://127.0.0.1:5173](http://127.0.0.1:5173)

---

### Запуск как десктоп-приложение (режим разработки)

Открывает окно Electron с горячей перезагрузкой:

```bash
npm run electron:dev
```

---

### Сборка установщика (быстрый способ)

В папке `scripts/` лежат готовые скрипты сборки — запустите нужный в зависимости от вашей ОС:

#### Windows

Дважды кликните `scripts\build-win.bat` или запустите из терминала:

```bat
scripts\build-win.bat
```

Скрипт соберёт портативный `.exe` и скопирует его в папку `release\`.

#### macOS

Один раз дайте права на запуск, затем запустите:

```bash
chmod +x scripts/build-mac.sh
./scripts/build-mac.sh
```

Скрипт соберёт `.dmg` для x64 и Apple Silicon и скопирует его в папку `release/`.

> Сборка для каждой платформы возможна только на ней же: `.exe` собирается на Windows, `.dmg` — на macOS.

---

### Сборка .exe для Windows (вручную)

Собирает портативный `.exe` в папку `dist-electron/`:

```bash
npm run electron:build
```

---

### Сборка .dmg для macOS (вручную)

Собирает `.dmg` для x64 и Apple Silicon (arm64) в папку `dist-electron/`:

```bash
npm run electron:build:mac
```

> Сборка macOS возможна только на компьютере под управлением macOS.
